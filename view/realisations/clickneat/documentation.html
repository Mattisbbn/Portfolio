Structure de la Base de Données :

1. Restaurant
  • Clé primaire : id
  • Relations :
    • Un restaurant a plusieurs catégories (1-n)
    • Un restaurant a plusieurs tables (1-n)
    • Un restaurant a plusieurs commandes (1-n)
2. User
  • Clé primaire : id
  • Relations :
    • Un utilisateur peut être associé à un restaurant (1-1)
    • Un utilisateur peut avoir plusieurs articles dans son panier (1-n)
    • Un utilisateur peut passer plusieurs commandes (1-n)
3. Category
  • Clé primaire : id
  • Clé étrangère : restaurant_id → restaurant(id)
  • Relations :
    • Une catégorie contient plusieurs items (1-n)
4. Item
  • Clé primaire : id
  • Clé étrangère : category_id → category(id)
  • Relations :
    • Un item peut avoir plusieurs ingrédients (1-n)
    • Un item peut avoir plusieurs allergènes (1-n)
    • Un item peut être dans plusieurs paniers (1-n)
    • Un item peut être dans plusieurs commandes (via order_item) (1-n)
5. Ingredient
  • Clé primaire : id
  • Clé étrangère : item_id → item(id)
6. Allergen
  • Clé primaire : id
  • Clé étrangère : item_id → item(id)
7. Table
  • Clé primaire : id
  • Clé étrangère : restaurant_id → restaurant(id)
  • Relations :
    • Une table peut avoir plusieurs réservations (1-n)
8. Reservation
  • Clé primaire : id
  • Clé étrangère : table_id → table(id)
  • Relations :
    • Une réservation peut être associée à une commande (1-1)
9. Cart
  • Clé primaire : id
  • Clés étrangères :
    • item_id → item(id)
    • user_id → user(id)
10. Order
  • Clé primaire : id
  • Clés étrangères :
    • user_id → user(id)
    • restaurant_id → restaurant(id)
    • reservation_id → reservation(id)
  • Relations :
    • Une commande contient plusieurs order_items (1-n)
11. Order_item
  • Clé primaire : id
  • Clés étrangères :
    • order_id → order(id)
    • item_id → item(id)
